# Landfalls

This directory contains repo-local landfall definitions.

A landfall is a named arrival ritual. Run it when an agent needs to refresh a specific slice of reality before acting.

## Definitions

| Landfall | Purpose | When To Run |
| --- | --- | --- |
| `example-landfall.yaml` | Replace with purpose. | Replace with trigger. |

## Rules

- Keep landfalls narrow and named by purpose.
- Write results back to the repo's existing task, wiki, evidence, calendar, or payable surfaces.
- Stop for human-only actions: credentials, MFA, identity, bank/wire details, legal judgment, and payments.
- If evidence changes a responsibility, update the task list.
